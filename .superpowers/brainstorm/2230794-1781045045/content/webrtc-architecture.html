Drive 60FPS WebRTC 架构草图

目标：320×240、端到端真实 60FPS；后端只做信令，视频点对点传输。

推荐架构：aiortc 内嵌 WebRTC Track
车端 / 模拟车端

DriveApiBridge

从 cam/image_array 获取 320×240 帧

将最新帧送入 aiortc.VideoStreamTrack

控制与状态继续走现有 WebSocket

Web UI 后端

FastAPI 信令

新增 WebRTC offer / answer / ICE 信令

不转发媒体流，不承担视频编码压力

保留旧 MJPEG 作为兼容降级

浏览器 Drive 页面

RTCPeerConnection

接收点对点视频轨道并绑定到 <video>

用 requestVideoFrameCallback 统计真实展示 FPS

失败时自动重建连接或降级 MJPEG

视频流：车端 ⇄ 浏览器 WebRTC P2P　　信令：车端/浏览器 ⇄ FastAPI　　控制：沿用 /api/drive/ws

设计重点

  • 把视频链路从 JSON/base64/MJPEG 中解耦
  • 后端只负责协调，不成为 60FPS 瓶颈
  • 保留现有控制通道，避免控制被视频阻塞

关键前提

  • Vehicle 主循环必须能提供接近 60Hz 的新帧
  • PC 模拟车端先验收，真实车端另行验证性能
  • 需要新增 WebRTC 依赖与信令测试
